# Preterm Labour — Synthetic Baseline Model Card

⚠ **This model is a SYNTHETIC BASELINE. Not clinically valid.**

## What it is
A classifier trained on 600 programmatically-generated samples
(`numpy.random` with seed=42) using rule-based labels that loosely
mirror clinical priors. Training takes <1 second and produces a tiny
pickle (~312 KB).

## Why it exists
The real training pipeline under `models/preterm_labour/` requires:
- Multi-GB datasets (see the pipeline's README for the source)
- Credentials (Kaggle / PhysioNet / HuggingFace) we don't have in CI
- Weeks of ML engineering to fill in the `NotImplementedError` stubs in
  `src/components/{data_ingestion,data_validation,data_transformation,model_trainer}.py`

Until those are completed and a real pickle replaces this one, this
synthetic baseline lets the LangGraph agent receive ML adapter output
in `tool_results` so the integration path is exercised end-to-end.

## What its predictions mean
**Nothing clinical.** They reflect the random label distribution +
weak heuristic the synthesis script encoded — not real disease prevalence,
not real risk stratification, not real biomarker physiology.

## Inputs
- `gestational_age_weeks`
- `rms_emg`
- `mean_emg`
- `median_freq_emg`
- `peak_freq_emg`
- `contractions_per_min`
- `contraction_amplitude_mean`
- `interval_mean_s`
- `interval_cv`

## Outputs
One of: Term, PretermRisk

## How to replace with a real model
1. Implement the `NotImplementedError` stubs in
   `models/preterm_labour/src/components/`
2. Run `python -m src.pipeline.training_pipeline` from that dir
3. Run `python models/preterm_labour/export_runtime.py` to extract the
   runtime pickle in this exact format: `{"preprocessor": ..., "model": ...}`
4. Drop the resulting `model.pkl` into this directory
5. Push to HF Space via `scripts/deploy-hf.sh`

The runtime adapter ([src/ml/preterm_labour_adapter.py](../../../src/ml/preterm_labour_adapter.py))
won't change — it's already wired to the right specialty graph.

`scripts/train_synthetic_baselines.py` — regenerate any time without
losing real weights (this script only overwrites paths it created).
